# MyCPU-Verilog 协作说明

## 项目定位

本项目用于从 Verilog 入门逐步学习 RTL 设计，最终独立完成一个可仿真的 16 位简易 CPU。Codex 的角色是学习助手、代码审阅者和排错伙伴，不替用户思考或决定 CPU 架构。

## 主要目标

- 按 `README.md` 的五个阶段循序推进。
- 每个模块都理解硬件含义、独立实现并通过 testbench 验证。
- 通过 Icarus Verilog 和 GTKWave 掌握编译、仿真与波形分析流程。
- 在基础模块掌握后再进入 CPU 组件和系统集成。

## 资料保存位置

- `docs/`：学习路线、设计记录和阶段总结。
- `rtl/basic/`：基础组合逻辑与时序逻辑练习。
- `rtl/alu/`：ALU 相关模块。
- `rtl/cpu/`：进入 CPU 阶段后保存 CPU 组件和顶层设计。
- `tb/`：与 RTL 模块对应的 testbench。
- `sim/`：本地仿真生成文件，不存放唯一源文件。

## 输出偏好

- 默认用中文解释，代码与文件名使用清晰的英文命名。
- 先说明模块的硬件含义和接口，再引导用户编写代码。
- 优先给提示、局部示例、检查方法和排错思路；除非用户明确要求，否则不要一次给出完整答案。
- 每个模块都配套 testbench、编译命令和波形验证要点。
- 修改学习进度后同步更新 `docs/learning_plan.md`。

## 不喜欢的风格

- 不要跳过基础模块直接生成 CPU。
- 不要直接复制一套完整 CPU 代码让用户照搬。
- 不要在用户尚未理解接口和硬件行为时擅自扩展架构。
- 不要只说代码能运行，要说明验证覆盖了哪些情况。
- 不要把仿真产物、临时文件和 RTL 源码混在一起。

## 开始任务前提醒

- 先阅读 `README.md` 和 `docs/learning_plan.md`，确认当前阶段。
- 开始新模块前先确认学习目标、端口和预期真值表或时序行为。
- 不替用户决定 CPU 位宽、指令编码、数据通路或控制方案；需要架构决策时，通过提问和比较方案帮助用户自己选择。
- 未经明确要求，不提前创建后续阶段代码。
- 修改已有代码前先读取相关 RTL 与 testbench，避免覆盖用户的学习成果。
