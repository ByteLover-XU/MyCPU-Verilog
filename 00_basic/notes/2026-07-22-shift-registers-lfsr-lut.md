# Verilog 学习笔记：移位寄存器、LFSR 与 LUT

日期：2026-07-22

## 一、总体说明

今天继续学习 HDLBits 的时序逻辑电路，主线从普通移位寄存器逐步扩展到循环移位、算术移位、线性反馈移位寄存器（LFSR），最后学习了如何用移位寄存器构造一个三输入查找表（LUT）。

今天涉及的主要内容如下：

- 带异步复位、同步装载和使能端的 4 位右移寄存器
- `if—else if` 所表达的控制优先级
- 100 位左、右循环移位寄存器
- 拼接运算符在循环移位中的使用
- 64 位算术移位寄存器
- 逻辑右移与算术右移的区别
- 5 位、3 位和 32 位伽罗瓦型 LFSR
- 根据电路图写出每一位寄存器的下一状态
- 使用串行写入、随机读取的 8 位移位寄存器实现三输入 LUT

今天最重要的主线是：

```text
先确定控制信号的优先级
        ↓
再根据当前状态写出下一状态
        ↓
最后在时钟有效沿用非阻塞赋值保存下一状态
```

无论是普通移位、循环移位、算术移位还是 LFSR，本质上都在回答同一个问题：

> 下一个时钟上升沿之后，寄存器的每一位应该保存什么？

## 二、题目、知识点与过程中出现的问题

### 题目一：4 位右移寄存器

题目要求设计一个 4 位右移寄存器，具有以下控制信号：

- `areset=1`：异步复位，立即将 `q` 清零。
- `load=1`：在下一个时钟上升沿把 `data` 装入 `q`。
- `ena=1`：在下一个时钟上升沿右移一位。
- `load=0` 且 `ena=0`：保持原值。

控制优先级为：

```text
areset > load > ena > 保持原值
```

正确代码：

```verilog
module top_module (
    input clk,
    input areset,
    input load,
    input ena,
    input [3:0] data,
    output reg [3:0] q
);

    always @(posedge clk or posedge areset) begin
        if (areset)
            q <= 4'b0000;
        else if (load)
            q <= data;
        else if (ena)
            q <= q >> 1;
    end

endmodule
```

右移也可以使用拼接运算符表示：

```verilog
q <= {1'b0, q[3:1]};
```

例如：

```text
移位前：q = 1011
移位后：q = 0101
```

#### `else if` 是否从上到下执行

`if—else if—else` 会从上到下判断条件。一旦某个条件成立，后面的分支就不再被选择。

例如 `load=1`、`ena=1` 时：

```verilog
else if (load)
    q <= data;
else if (ena)
    q <= q >> 1;
```

只有装载分支生效，因此 `q` 装入 `data`，不会同时右移。

更准确地说，综合后的硬件形成了带优先级的选择电路，而不是像普通软件一样一句一句花费时间执行。

#### 过程中出现的问题：使用三个独立的 `if`

错误思路：

```verilog
if (areset)
    q <= 0;
if (load && ~ena)
    q <= data;
if (ena)
    q <= q >> 1;
```

三个独立的 `if` 可能在同一次触发中同时成立，不能正确表达题目规定的优先级。尤其当 `load=1`、`ena=1` 时，题目要求装载优先，但最后一个赋值可能让右移结果覆盖前面的安排。

正确方法是使用一条 `if—else if` 链。由于前面的 `load` 分支已经具有更高优先级，所以不需要写：

```verilog
if (load && ~ena)
```

### 题目二：100 位左、右循环移位寄存器

循环移位不会丢弃从一端移出的位，而是把它送回另一端。

题目规定：

| `ena` | 操作 |
|---|---|
| `2'b01` | 向右循环移动 1 位 |
| `2'b10` | 向左循环移动 1 位 |
| `2'b00` | 保持原值 |
| `2'b11` | 保持原值 |

`load` 是同步装载，并且优先于循环移位。

正确代码：

```verilog
module top_module (
    input clk,
    input load,
    input [1:0] ena,
    input [99:0] data,
    output reg [99:0] q
);

    always @(posedge clk) begin
        if (load)
            q <= data;
        else begin
            case (ena)
                2'b01: q <= {q[0], q[99:1]};
                2'b10: q <= {q[98:0], q[99]};
                default: q <= q;
            endcase
        end
    end

endmodule
```

在时序逻辑中，`default` 分支也可以不对 `q` 赋值，此时寄存器自然保持原值。

#### 右循环的位移动关系

以 4 位数据 `1001` 为例：

```text
位编号：     3 2 1 0
循环前：     1 0 0 1
普通右移：   0 1 0 0
右循环后：   1 1 0 0
```

右侧移出的旧 `q[0]` 回到最左侧的新 `q[3]`：

```verilog
q <= {q[0], q[3:1]};
```

推广到 100 位：

```verilog
q <= {q[0], q[99:1]};
```

#### 左循环的位移动关系

```text
循环前：     1 0 0 1
普通左移：   0 0 1 0
左循环后：   0 0 1 1
```

左侧移出的旧 `q[3]` 回到最右侧的新 `q[0]`：

```verilog
q <= {q[2:0], q[3]};
```

推广到 100 位：

```verilog
q <= {q[98:0], q[99]};
```

#### 过程中出现的问题：循环移位补回的位置写反

一开始把右移后补回的位置写成了 `q[0]`，把左移后补回的位置写成了 `q[99]`。问题来自混淆了“左右方向”和“位编号”。

最稳妥的检查方法是拿 `1001` 这样的 4 位数据手动移动一次。最终规律是：

```text
右循环：旧 q[0]  → 新 q[99]
左循环：旧 q[99] → 新 q[0]
```

### 题目三：64 位算术移位寄存器

题目要求根据 `amount` 选择移位方向和位数：

| `amount` | 操作 |
|---|---|
| `2'b00` | 左移 1 位 |
| `2'b01` | 左移 8 位 |
| `2'b10` | 算术右移 1 位 |
| `2'b11` | 算术右移 8 位 |

控制优先级：

```text
load > ena > 保持原值
```

正确代码：

```verilog
module top_module (
    input clk,
    input load,
    input ena,
    input [1:0] amount,
    input [63:0] data,
    output reg [63:0] q
);

    always @(posedge clk) begin
        if (load)
            q <= data;
        else if (ena) begin
            case (amount)
                2'b00: q <= q << 1;
                2'b01: q <= q << 8;
                2'b10: q <= $signed(q) >>> 1;
                2'b11: q <= $signed(q) >>> 8;
            endcase
        end
    end

endmodule
```

#### 逻辑右移与算术右移

逻辑右移左侧补 `0`：

```verilog
q >> 1
```

算术右移左侧补原来的符号位 `q[63]`：

```verilog
$signed(q) >>> 1
```

例如对 8 位数据：

```text
原数据：        10110000
逻辑右移 1 位： 01011000
算术右移 1 位： 11011000
```

最高位为 `1` 时，算术右移持续补 `1`，从而保持有符号数的负号。算术左移与逻辑左移的补位方式相同，右侧都补 `0`。

`$signed(q)` 只是在这个表达式中临时把 `q` 解释成有符号数，真正表示算术右移的运算符是 `>>>`。

#### 过程中出现的问题一：装载了错误的信号

错误：

```verilog
if (load)
    q <= load;
```

`load` 只是 1 位控制信号，题目要求装入的是 64 位输入 `data`：

```verilog
if (load)
    q <= data;
```

#### 过程中出现的问题二：只写表达式，没有保存结果

错误：

```verilog
$signed(q) >>> 1;
```

这只计算了一个表达式，却没有把结果写入任何寄存器。应当写成：

```verilog
q <= $signed(q) >>> 1;
```

### 题目四：5 位伽罗瓦型 LFSR

线性反馈移位寄存器（LFSR）使用当前寄存器中的某一位作为反馈位，并在指定抽头处进行异或运算，以产生下一个状态。

一个选择正确抽头的 `n` 位最大长度 LFSR 可以在重复之前经历：

$$
2^n-1
$$

种非零状态。

5 位最大长度 LFSR 可以经历：

$$
2^5-1=31
$$

种非零状态。全零状态经过移位和异或后仍然是全零，因此会被锁死，复位时应装入非零值。本题要求复位为十进制 `1`，即 `5'b00001`。

根据图中的连线，各位下一状态为：

```verilog
q[4] <= q[0];
q[3] <= q[4];
q[2] <= q[3] ^ q[0];
q[1] <= q[2];
q[0] <= q[1];
```

可以合并为：

```verilog
module top_module (
    input clk,
    input reset,
    output reg [4:0] q
);

    always @(posedge clk) begin
        if (reset)
            q <= 5'b00001;
        else
            q <= {q[0], q[4], q[3] ^ q[0], q[2], q[1]};
    end

endmodule
```

这里不能简单写成普通整体右移，因为新 `q[2]` 还要经过异或门。

### 题目五：带同步并行装载的 3 位 LFSR

题目给出一张包含三个 MUX 和三个 D 触发器的电路图，并规定开发板端口对应关系：

```text
SW[2:0]   → r2、r1、r0
KEY[0]    → Clock
KEY[1]    → L
LEDR[2:0] → Q2、Q1、Q0
```

当 `L=1` 时，在下一个时钟上升沿并行装入 `SW`：

```text
新 Q0 = r0
新 Q1 = r1
新 Q2 = r2
```

当 `L=0` 时，按照反馈关系更新：

```text
新 Q0 = 旧 Q2
新 Q1 = 旧 Q0
新 Q2 = 旧 Q1 XOR 旧 Q2
```

可以直接使用向量写法：

```verilog
module top_module (
    input [2:0] SW,
    input [1:0] KEY,
    output reg [2:0] LEDR
);

    always @(posedge KEY[0]) begin
        if (KEY[1])
            LEDR <= SW;
        else
            LEDR <= {LEDR[1] ^ LEDR[2], LEDR[0], LEDR[2]};
    end

endmodule
```

也可以先把三个 MUX 的组合逻辑分别写出来：

```verilog
wire d0, d1, d2;

assign d0 = KEY[1] ? SW[0] : LEDR[2];
assign d1 = KEY[1] ? SW[1] : LEDR[0];
assign d2 = KEY[1] ? SW[2] : (LEDR[2] ^ LEDR[1]);

always @(posedge KEY[0]) begin
    LEDR[0] <= d0;
    LEDR[1] <= d1;
    LEDR[2] <= d2;
end
```

#### 过程中出现的问题一：MUX 选择信号接错

曾把选择条件写成：

```verilog
assign d0 = SW[1] ? ...;
```

但题目规定控制信号 `L` 连接到 `KEY[1]`，因此三个 MUX 都必须由 `KEY[1]` 控制。

#### 过程中出现的问题二：`LEDR` 的类型不正确

`LEDR` 在 `always` 块中被赋值，因此在 Verilog 中必须声明为：

```verilog
output reg [2:0] LEDR;
```

另外，定义后没有参与任何逻辑的 `q0`、`q1`、`q2` 可以删除。

### 题目六：32 位伽罗瓦型 LFSR

题目要求构建一个 32 位伽罗瓦型 LFSR，抽头位于第 32、22、2、1 位，并使用高电平有效的同步复位。

抽头编号与 Verilog 位编号的对应关系：

```text
第 32 位 → q[31]
第 22 位 → q[21]
第 2 位  → q[1]
第 1 位  → q[0]
```

复位值为：

```verilog
32'h00000001
```

下一状态可以分段拼接：

```verilog
module top_module (
    input clk,
    input reset,
    output reg [31:0] q
);

    always @(posedge clk) begin
        if (reset)
            q <= 32'h00000001;
        else begin
            q <= {
                q[0],
                q[31:23],
                q[22] ^ q[0],
                q[21:3],
                q[2] ^ q[0],
                q[1] ^ q[0]
            };
        end
    end

endmodule
```

由于 `reset` 是同步复位，敏感列表中只写：

```verilog
always @(posedge clk)
```

不能添加 `posedge reset`。

写这种长向量拼接时，应检查每一段的总位宽。本题各段宽度之和为：

```text
1 + 9 + 1 + 19 + 1 + 1 = 32
```

### 题目七：用移位寄存器实现三输入 LUT

题目首先要求使用 8 个 D 触发器构建一个 8 位移位寄存器：

```text
S → Q[0] → Q[1] → Q[2] → ... → Q[7]
```

当 `enable=1` 时，每个时钟上升沿执行一次串行移位：

```verilog
Q[0] <= S;
Q[1] <= Q[0];
Q[2] <= Q[1];
// ...
Q[7] <= Q[6];
```

合并为向量赋值：

```verilog
q <= {q[6:0], S};
```

`MSB is shifted in first` 表示写入一个 8 位数据时，先输入 `data[7]`，最后输入 `data[0]`。连续移入 8 位后，最终各位正好保存：

```text
q[7:0] = data[7:0]
```

然后使用 `A、B、C` 组成三位地址，随机读取 `q` 中的一位：

| `ABC` | `Z` |
|---|---|
| `000` | `q[0]` |
| `001` | `q[1]` |
| `010` | `q[2]` |
| `011` | `q[3]` |
| `100` | `q[4]` |
| `101` | `q[5]` |
| `110` | `q[6]` |
| `111` | `q[7]` |

完整代码：

```verilog
module top_module (
    input clk,
    input enable,
    input S,
    input A,
    input B,
    input C,
    output Z
);

    reg [7:0] q;

    always @(posedge clk) begin
        if (enable)
            q <= {q[6:0], S};
    end

    assign Z = q[{A, B, C}];

endmodule
```

这里包含两类逻辑：

- 写入部分是时序逻辑，必须等待 `clk` 上升沿。
- 读取部分是组合逻辑，地址 `ABC` 改变后，`Z` 随所选寄存器位变化。

题目没有复位信号，所以开始仿真后应连续写入 8 位，再认为整个寄存器中的内容全部有效。

## 三、总结与反思

今天学习的几类电路虽然功能不同，但都可以用“当前状态到下一状态”的方法统一分析：

```text
普通移位寄存器：相邻位之间传递数据，空位补 0
循环移位寄存器：移出的位从另一端重新进入
算术右移寄存器：右移时复制符号位
LFSR：移位过程中在抽头位置加入反馈异或
LUT：先把真值表串行写入寄存器，再用输入作为地址读取一位
```

今天需要重点巩固以下内容：

- `if—else if` 表示从上到下的优先级；独立的 `if` 表示条件可能同时成立。
- 异步复位需要出现在敏感列表中，同步复位只在时钟有效沿检查。
- 时序逻辑使用非阻塞赋值 `<=`，未赋值的寄存器保持原值。
- 右循环时旧最低位回到新最高位，左循环时旧最高位回到新最低位。
- `>>` 是逻辑右移，`$signed(q) >>>` 才能让无符号声明的向量按有符号数进行算术右移。
- 一个移位表达式必须赋给寄存器，单独计算表达式不会改变电路状态。
- LFSR 不能只看成普通移位，必须逐位检查抽头和异或关系。
- 端口名称和开发板连线必须按照题意对应，不能把数据输入误当成选择信号。
- 长向量拼接要同时检查顺序与总位宽。
- LUT 的串行写入是时序逻辑，地址读取是组合逻辑。

今天暴露出的主要问题：

- 对多个控制信号同时有效时的优先级还不够敏感。
- 容易把循环移位的补回方向写反。
- 容易只写运算表达式而忘记把结果赋回寄存器。
- 阅读 LFSR 电路图时，需要更仔细地区分当前位和下一状态位。
- 根据开发板端口写代码时，容易把 `SW` 和 `KEY` 的功能混淆。

以后遇到移位寄存器或 LFSR 题目，可以按照下面的顺序处理：

```text
确认时钟边沿和复位类型
        ↓
列出 load、enable 等控制信号的优先级
        ↓
标清最高位、最低位和移位方向
        ↓
逐位写出“新 q[i] = 旧的什么”
        ↓
确认抽头、反馈位和异或位置
        ↓
用拼接运算符合并
        ↓
检查总位宽、output reg 和非阻塞赋值
```

今天最需要形成的思维是：

> 不要先凭感觉写 `<<` 或 `>>`，而要先明确每一个旧位在下一个时钟沿后去了哪里，再把这个位移动关系翻译成 Verilog。
